# 🚀 IT 기초 용어 정리 가이드

웹과 앱의 차이부터 프론트엔드/백엔드의 개념까지 핵심만 콕 짚어 정리해 드립니다.

---

## 1. 플랫폼의 종류: 웹, 앱, 그리고 웹앱

사용자가 서비스에 어떻게 접속하느냐에 따라 구분됩니다.

* **웹 (Web):** 브라우저(크롬, 사파리 등)를 통해 접속하는 페이지입니다. 설치 없이 URL만으로 접근 가능합니다.
* **앱 (Application):** 스토어에서 다운로드하여 스마트폰에 직접 **설치**하는 프로그램입니다. 기기의 성능을 최대로 활용할 수 있습니다.
* **웹앱 (WebApp):** 브라우저에서 실행되지만, 디자인이나 동작 방식이 **앱처럼 느껴지도록** 만든 웹사이트입니다.

---

## 2. 랜딩페이지 (Landing Page)

사용자가 광고나 링크를 클릭했을 때 **가장 먼저 도착(Landing)하는 페이지**를 말합니다.

* **핵심 목표:** 정보 전달보다는 **특정 행동 유도**(회원가입, 구매하기, 구독하기 등)에 집중합니다. 보통 군더더기 없는 깔끔한 디자인과 강력한 문구로 구성됩니다.

---

## 3. 프론트엔드와 백엔드 (Frontend & Backend)

서비스를 하나의 식당이라고 생각하면 이해가 쉽습니다.

### 🎨 프론트엔드 (Frontend) - "식당의 홀"
사용자가 눈으로 보고 직접 만지는 **사용자 인터페이스(UI)** 영역입니다.
* **비유:** 메뉴판, 테이블 디자인, 조명, 서빙 서비스.
* **역할:** 버튼의 위치, 글꼴, 애니메이션 등 화면상의 모든 시각적인 요소를 구현합니다.

### ⚙️ 백엔드 (Backend) - "식당의 주방"
사용자의 눈에 보이지 않는 곳에서 데이터를 처리하고 저장하는 **서버와 데이터베이스** 영역입니다.
* **비유:** 요리사, 냉장고(식재료 보관), 주문 관리 시스템.
* **역할:** 회원가입 시 중복 아이디 체크, 결제 승인 처리, 게시글 저장 등 시스템의 핵심 로직을 담당합니다.

---

| 구분 | 프론트엔드 (Frontend) | 백엔드 (Backend) |
| :--- | :--- | :--- |
| **핵심 키워드** | UI/UX, 화면 구현 | 서버, 데이터베이스, 로직 |
| **주요 언어** | HTML, CSS, JavaScript | Java, Python, Node.js |
| **사용자 관계** | 직접 대면 (사용자 경험 중심) | 간접 연결 (데이터 처리 중심) |
